# Merge Intervals

The **Merge Intervals** pattern is a highly specific but incredibly common algorithm pattern. It deals with overlapping segments, whether they represent meeting times, scheduling appointments, or purely numerical ranges.

If a problem mentions "overlapping", "intervals", "schedules", or "ranges", there is a 99% chance you need this pattern.

## The Core Concept

An interval is simply an array or pair of two numbers: `[start, end]`. 
For example, a meeting that starts at 1:00 PM and ends at 3:00 PM could be represented as `[1, 3]`.

The fundamental rule of overlapping intervals is:
**Two intervals `A` and `B` overlap if `A.start <= B.end` AND `B.start <= A.end`.**

However, checking every interval against every other interval takes `O(N²)`. We can optimize this drastically.

## Step 1: Sort by Start Time

The absolute secret to solving almost *any* interval problem is to **sort the intervals by their start time** first. 

Once sorted, any overlapping intervals will be adjacent to each other in the array. This reduces the problem from a complex geometric puzzle into a simple, linear pass-through!

## Step 2: Merge the Adjacent Intervals

Once sorted, you can iterate through the array. For each new interval, compare it only to the *last* interval you added to your merged list.

```javascript
function mergeIntervals(intervals) {
  if (intervals.length < 2) return intervals;

  // 1. Sort by start time
  intervals.sort((a, b) => a[0] - b[0]);

  const merged = [intervals[0]];

  for (let i = 1; i < intervals.length; i++) {
    const current = intervals[i];
    const lastMerged = merged[merged.length - 1];

    // 2. Check for overlap
    if (current[0] <= lastMerged[1]) {
      // They overlap! Update the end time to be the maximum
      lastMerged[1] = Math.max(lastMerged[1], current[1]);
    } else {
      // They don't overlap. Add the current interval.
      merged.push(current);
    }
  }

  return merged;
}
```

## Time Complexity

Because sorting the array is the most expensive operation, the time complexity of the Merge Intervals pattern is universally bounded by **`O(N log N)`**, where `N` is the number of intervals.

The merging pass itself is only `O(N)`.

## Wrapping Up

Whenever you see a problem involving time spans, immediately sort the input. Sorting turns chaotic, overlapping timelines into a neat, linear sequence that you can process in a single sweep!
